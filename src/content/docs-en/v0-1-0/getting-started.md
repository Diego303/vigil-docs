# Getting Started

Sentinel is a deterministic security scanner specifically designed for code generated by AI agents. Unlike tools like Semgrep or Bandit, Sentinel focuses on detecting patterns exclusive to LLM-generated code: hallucinated dependencies, empty tests, and permissive security configurations.

This guide will take you from zero to your first scan in under 5 minutes.

## Prerequisites

Before you begin, make sure you have:

- **Python 3.9** or higher
- **pip** up to date (`pip install --upgrade pip`)
- **Git** (optional, needed for `--changed-only`)
- Terminal access on your operating system

Verify your Python version:

```bash
python --version
# Python 3.11.4
```

## Quick Install

Install Sentinel directly from PyPI:

```bash
pip install sentinel-ai
```

Verify the installation:

```bash
sentinel --version
# sentinel v0.1.0

sentinel --help
# Usage: sentinel [OPTIONS] COMMAND [ARGS]...
#
# Commands:
#   scan     Scan a directory for vulnerabilities
#   init     Create a .sentinel.yaml with default configuration
#   rules    List all available rules
#   version  Show current version
```

## Your First Scan

### Basic scan

Run Sentinel on your source code directory:

```bash
sentinel scan src/
```

Sentinel will recursively analyze all supported files (`.py`, `.js`, `.ts`, `requirements.txt`, `package.json`, etc.) and display a human-readable report.

### Example output

```bash
$ sentinel scan src/ --format human

◇ Sentinel v0.1.0 — Scanning 42 files...
==================================================

[CRITICAL] DEP-001: Dependency Hallucination
  › File: requirements.txt:14
  › Package: 'fastapi-auth-middleware' DOES NOT EXIST on PyPI.
  › Risk: High potential for Slopsquatting.
  › Suggestion: Verify the package name or remove it.

[ALERT] TEST-001: Test Theater Detected
  › File: tests/test_auth.py:22
  › Function: 'test_verify_token' passes but contains no valid asserts.
  › Suggestion: Add at least one assert that validates the result.

[ALERT] SEC-002: Permissive CORS
  › File: src/main.py:8
  › Detail: allow_origins=["*"] allows access from any domain.
  › Suggestion: Restrict origins to known domains.

==================================================
✗ Scan failed: 3 issues found in 1.4s
  › 1 critical, 2 alerts, 0 info
```

### Interpreting results

Each finding includes:

| Field | Description |
|-------|-------------|
| **Severity** | `[CRITICAL]`, `[ALERT]`, or `[INFO]` |
| **Rule** | Unique identifier (e.g. `DEP-001`) |
| **File** | Exact location with line number |
| **Detail** | Description of the issue found |
| **Suggestion** | Recommended action to resolve it |

## Output Formats

Sentinel supports three output formats to adapt to different workflows:

| Format | Flag | Typical use |
|--------|------|-------------|
| Human | `--format human` | Direct terminal reading |
| JSON | `--format json` | Integration with scripts and pipelines |
| SARIF | `--format sarif` | GitHub Code Scanning and IDEs |

### Export to JSON

```bash
sentinel scan src/ --format json --output report.json
```

```json
{
  "version": "0.1.0",
  "scan": {
    "files_scanned": 42,
    "duration_ms": 1400,
    "status": "fail"
  },
  "findings": [
    {
      "rule": "DEP-001",
      "severity": "critical",
      "file": "requirements.txt",
      "line": 14,
      "message": "Package 'fastapi-auth-middleware' does not exist on PyPI",
      "suggestion": "Verify the package name or remove it"
    }
  ]
}
```

### Export to SARIF (GitHub)

```bash
sentinel scan src/ --format sarif --output report.sarif
```

Upload the SARIF report directly to GitHub Code Scanning:

```yaml
# .github/workflows/sentinel.yml
- name: Upload SARIF
  uses: github/codeql-action/upload-sarif@v3
  with:
    sarif_file: report.sarif
```

## Initialize Configuration

To create a `.sentinel.yaml` file with default configuration:

```bash
sentinel init
# ✓ Created .sentinel.yaml with default configuration
```

This generates a preconfigured file you can customize for your project's needs.

## Pre-commit Integration

Add Sentinel as a pre-commit hook to automatically scan each commit:

```yaml
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/Diego303/sentinel-cli
    rev: v0.1.0
    hooks:
      - id: sentinel
        args: [scan, --changed-only, --severity, warning]
```

The `--changed-only` flag ensures only files modified in the commit are scanned, keeping the hook fast.
