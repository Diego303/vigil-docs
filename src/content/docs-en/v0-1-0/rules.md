# Detection Rules

Vigil organizes its rules into three main categories. Each rule has a unique identifier, an assigned severity, and configuration options.

The categories are designed to cover the most common attack vectors introduced by AI-generated code:

| Category | Prefix | Focus |
|----------|--------|-------|
| **Dependencies** | `DEP-` | Hallucinated packages, typosquatting |
| **Security** | `SEC-` | Excessive permissions, exposed secrets |
| **Tests** | `TEST-` | Fake tests, artificial coverage |

---

## DEP — Dependencies

Rules that verify the existence and legitimacy of project dependencies. These rules are the primary defense against **slopsquatting**.

### DEP-001: Dependency Hallucination

| Property | Value |
|----------|-------|
| **Severity** | Critical |
| **Autofix** | No |
| **Files** | `requirements.txt`, `package.json`, `Cargo.toml`, `go.mod` |

**Description**: Detects packages that don't exist in the corresponding registry. LLMs frequently invent plausible-sounding package names that don't exist. An attacker can register that name on the registry and execute arbitrary code on the developer's machine.

**Example of vulnerable code**:

```python
# requirements.txt — generated by an LLM
flask==3.0.0
flask-sqlalchemy==3.1.1
fastapi-auth-middleware==1.0.0    # ← DOES NOT EXIST on PyPI
python-jwt-validator==2.3.0       # ← DOES NOT EXIST on PyPI
requests==2.31.0
```

**Vigil output**:

```bash
[CRITICAL] DEP-001: Dependency Hallucination
  › File: requirements.txt:3
  › Package: 'fastapi-auth-middleware' DOES NOT EXIST on PyPI.
  › Risk: Slopsquatting — an attacker can register this name.
  › Suggestion: Verify the correct name. Did you mean 'fastapi-users'?

[CRITICAL] DEP-001: Dependency Hallucination
  › File: requirements.txt:4
  › Package: 'python-jwt-validator' DOES NOT EXIST on PyPI.
  › Suggestion: Real alternatives: 'PyJWT', 'python-jose', 'authlib'.
```

**Configuration**:

```yaml
rules:
  DEP-001:
    enabled: true
    severity: critical
    options:
      registries: [pypi, npm, crates.io]  # Registries to verify
      cache_ttl: 3600                      # Cache in seconds
      suggest_alternatives: true           # Suggest similar packages
```

### DEP-002: New Package Alert

| Property | Value |
|----------|-------|
| **Severity** | Warning |
| **Autofix** | No |
| **Files** | `requirements.txt`, `package.json`, `Cargo.toml` |

**Description**: Alerts on dependencies published less than 30 days ago. New packages carry higher risk of being malicious, especially if they have names similar to popular packages.

**Example detection**:

```bash
[ALERT] DEP-002: New Package Alert
  › File: package.json:12
  › Package: 'react-auth-helper' published 3 days ago on npm.
  › Total downloads: 47
  › Suggestion: Manually verify the package before using it.
```

**Configuration**:

```yaml
rules:
  DEP-002:
    enabled: true
    severity: warning
    options:
      max_age_days: 30        # Alert if package is less than N days old
      min_downloads: 100      # Minimum download threshold
```

## SEC — Security

Rules that detect deficient security configurations introduced by AI-generated code. LLMs tend to prioritize making code "work" over making it secure.

### SEC-001: Over-Permission

| Property | Value |
|----------|-------|
| **Severity** | Critical |
| **Autofix** | No |
| **Files** | `.py`, `.js`, `.ts` |

**Description**: Detects sensitive HTTP endpoints (admin, payments, user management) lacking authentication or authorization middleware.

**Example of vulnerable code**:

```python
# src/routes.py — generated by AI without protection
from fastapi import FastAPI

app = FastAPI()

@app.get("/api/users")
async def list_users():                    # ← No Auth middleware
    return db.get_all_users()

@app.delete("/api/users/{user_id}")
async def delete_user(user_id: int):       # ← No Auth middleware
    return db.delete_user(user_id)

@app.get("/api/admin/settings")
async def admin_settings():               # ← No Auth middleware
    return db.get_settings()
```

**Fixed code**:

```python
from fastapi import FastAPI, Depends
from app.auth import require_auth, require_admin

app = FastAPI()

@app.get("/api/users")
async def list_users(user=Depends(require_auth)):
    return db.get_all_users()

@app.delete("/api/users/{user_id}")
async def delete_user(user_id: int, user=Depends(require_admin)):
    return db.delete_user(user_id)
```

**Configuration**:

```yaml
rules:
  SEC-001:
    enabled: true
    options:
      sensitive_paths:
        - "/api/admin/*"
        - "/api/users/*"
        - "/api/payments/*"
        - "/api/settings/*"
```

### SEC-002: Permissive CORS

| Property | Value |
|----------|-------|
| **Severity** | Warning |
| **Autofix** | No |
| **Files** | `.py`, `.js`, `.ts` |

**Description**: Detects CORS configurations with `allow_origins=["*"]` or `Access-Control-Allow-Origin: *`. This allows any website to make requests to your API, opening the door to CSRF attacks.

**Example of vulnerable code**:

```python
from fastapi.middleware.cors import CORSMiddleware

# AI generates this to "make it work quickly"
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],           # ← Permissive
    allow_credentials=True,        # ← Dangerous with origins=*
    allow_methods=["*"],
    allow_headers=["*"],
)
```

**Fixed code**:

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "https://app.example.com",
        "https://admin.example.com",
    ],
    allow_credentials=True,
    allow_methods=["GET", "POST", "PUT", "DELETE"],
    allow_headers=["Authorization", "Content-Type"],
)
```

### SEC-003: Hardcoded Secrets

| Property | Value |
|----------|-------|
| **Severity** | Critical |
| **Autofix** | No |
| **Files** | `.py`, `.js`, `.ts`, `.env`, `.yaml`, `.json` |

**Description**: Detects hardcoded secrets or placeholder values that AI copies from examples and `.env.example` without replacing them with environment variables.

**Detected patterns**:

```python
# All of these are detected by SEC-003:

JWT_SECRET = "your-secret-key-here"        # Placeholder
API_KEY = "sk-1234567890abcdef"            # Real credential
DATABASE_URL = "postgres://user:pass@localhost/db"  # Credential in URL
SECRET_KEY = "changeme"                     # Default value
AWS_ACCESS_KEY = "AKIA1234567890EXAMPLE"   # AWS key
```

**Fixed code**:

```python
import os

JWT_SECRET = os.environ["JWT_SECRET"]
API_KEY = os.environ["API_KEY"]
DATABASE_URL = os.environ["DATABASE_URL"]
```

**Configuration**:

```yaml
rules:
  SEC-003:
    enabled: true
    options:
      entropy_threshold: 3.5    # Shannon entropy minimum
      patterns:
        - "your-.*-here"
        - "TODO"
        - "CHANGEME"
        - "example"
        - "placeholder"
```

## TEST — Tests

Rules that detect tests that don't actually test anything. LLMs generate tests that create the illusion of coverage without validating behavior.

### TEST-001: Test Theater

| Property | Value |
|----------|-------|
| **Severity** | Warning |
| **Autofix** | No |
| **Files** | `test_*.py`, `*.test.js`, `*.test.ts`, `*.spec.*` |

**Description**: Detects test functions that contain no valid asserts. These tests always pass, artificially inflating coverage metrics.

**Example of empty test (detected)**:

```python
def test_verify_token():
    """Test generated by AI — looks good but tests nothing."""
    token = create_token("user@test.com")
    result = verify_token(token)
    # ← No assert. Test always passes.

def test_create_user():
    user = User(name="Test", email="test@test.com")
    assert user is not None      # ← Useless assert: never None
```

**Fixed test**:

```python
def test_verify_token():
    token = create_token("user@test.com")
    result = verify_token(token)
    assert result.valid is True
    assert result.email == "user@test.com"
    assert result.expires_at > datetime.utcnow()

def test_create_user():
    user = User(name="Test", email="test@test.com")
    assert user.name == "Test"
    assert user.email == "test@test.com"
    assert user.id is not None
```

**Configuration**:

```yaml
rules:
  TEST-001:
    enabled: true
    options:
      min_asserts: 1          # Minimum asserts per test
      ignore_patterns:
        - "test_smoke_*"      # Ignore smoke tests
```

### TEST-002: Mirror Mock

| Property | Value |
|----------|-------|
| **Severity** | Info |
| **Autofix** | No |
| **Files** | `test_*.py`, `*.test.js`, `*.test.ts` |

**Description**: Detects tests where the mock exactly replicates the real implementation. This creates a circular test that can never fail.

**Example of mirror mock (detected)**:

```python
def calculate_discount(price, percentage):
    return price * (percentage / 100)

# Mock replicates the same logic — circular test
@patch('app.pricing.calculate_discount')
def test_discount(mock_calc):
    mock_calc.side_effect = lambda p, pct: p * (pct / 100)  # ← Mirror
    result = apply_discount(100, 20)
    assert result == 80   # Always passes because mock IS the implementation
```

## Disabling Rules

### In configuration

```yaml
rules:
  TEST-002:
    enabled: false
```

### On command line

```bash
vigil scan src/ --ignore DEP-002 --ignore TEST-002
```

### Inline (per file)

Add a comment to ignore a specific line:

```python
JWT_SECRET = "dev-only-secret"  # vigil-ignore: SEC-003
```
